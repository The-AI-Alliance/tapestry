# Governance

This directory contains governance design documents for Project Tapestry. Governance is treated as an architectural concern — these constraints are load-bearing for the system design, not policy afterthoughts.

Consortium participants are assumed to be **collaborating institutions**, not mutually adversarial parties by default. Legal and organizational agreements are valid sovereignty mechanisms alongside technical controls. See [Design principles for architecture work](../architecture/0-tva-methodology.md#design-principles-for-architecture-work).
