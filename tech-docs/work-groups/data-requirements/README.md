# README for the Data Requirements Work Group

Identify and prioritize the data sovereignty, governance, and management requirements for the "spheres" of national, cultural, and industrial sovereignty.

## Key Questions

1. What specific data categories exist in each sovereignty sphere, and what legal constraints apply to each?
2. What metadata and provenance standards are needed to support consent tracking and data revocation?
3. How should the 70/30 architecture principle be reflected in data schema and pipeline design?
4. What are the minimum data residency requirements for each of the three sovereignty levels?
5. How is "sacred knowledge" operationalized as a technical data category, not just a policy designation?
6. What bias detection and cultural representation auditing is required at the data layer?
7. What public-sector data MOU templates are needed to make sovereign data access legally operable?
